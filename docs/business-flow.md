# 業務フロー（自分用問い合わせ管理）

## 登場人物

- 社内 SE（自分）
- ユーザー（将来的に拡張で登録可能）

## ステータス一覧

- open: 問い合わせ登録済み、対応未着手
- investigating: 対応中（調査・確認）
- pending: 回答待ち（ユーザー・他部署確認中）
- closed: 対応完了、結果記録済み

## 問い合わせ対応フロー（社内 SE 視点）

1. 社内 SE（自分）が問い合わせを受けたら、新しいチケットを登録（ステータス: open）
2. 調査・対応を行い、対応履歴を記録（ステータス: investigating）
3. 回答や対応が完了したら、結果を記録してステータスを closed に更新
4. 同じような問い合わせが将来あった場合、検索して参照

## 将来的な拡張

- ユーザーが直接問い合わせ登録可能にする
- 権限管理（閲覧・編集制御）
- SLA 管理や通知機能を追加
