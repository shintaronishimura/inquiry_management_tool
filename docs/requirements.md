# 要件定義（社内 SE 用問い合わせ管理ツール）

## 1. 本ツールの目的

- 社内 SE としての問い合わせ対応の属人化を防ぐ
- 自分の対応履歴を蓄積し、検索・参照できるようにする
- 将来的には社内ユーザーが直接登録可能にすることも視野

## 2. 登録する情報（問い合わせ）

- タイトル
- 内容
- 問い合わせ元（部署）
- ステータス（open / investigating / pending / closed）

## 3. 対応履歴として記録する情報

- 対応内容
- 対応者
- 対応日時

## 4. 完了時の情報

- 結果（対応完了時の状態やコメント）

## 5. 検索で利用する条件

- キーワード（タイトル・内容・履歴）
- ステータス
- 部署
